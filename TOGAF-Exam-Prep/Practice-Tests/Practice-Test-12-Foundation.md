## Answer Key

### Question 1
**Correct Answer: b)**
The Request for Architecture Work is a key trigger/input for Phase A. It comes from the sponsoring organization or from Phase H of a prior ADM cycle. The Architecture Definition Document is developed during and after Phase A, not an input to it.

### Question 2
**Correct Answer: c)**
The Technology Standards Catalog lists the agreed technology standards. Catalogs list building blocks of a specific type. The Application Communication Diagram is a diagram, and the Application/Technology Matrix is a matrix.

### Question 3
**Correct Answer: c)**
Consistent means the implementation has some architectural features on an unofficial basis—the implementation is moving in the right direction but without formal agreement or mandate. This is a commonly missed distinction on the exam.

### Question 4
**Correct Answer: a)**
Phase B outputs include a Refined Architecture Vision (updated based on detailed business architecture work), the Architecture Definition Document (business domain), and updated Architecture Requirements Specification. The Implementation and Migration Plan is a Phase F output.

### Question 5
**Correct Answer: d)**
The Architecture Sponsor provides the strategic business direction and funding authority. This role is distinct from the Enterprise Architect (who develops architecture) and the Architecture Board Member (who governs architecture decisions).

### Question 6
**Correct Answer: b)**
The Stakeholder Map Matrix is classified as a matrix. It maps stakeholders to their concerns and influence/power levels. Despite having "map" in its name, it is formally a matrix artifact in the TOGAF Content Framework.

### Question 7
**Correct Answer: b)**
Key inputs to the Preliminary Phase include the TOGAF Library, other architecture frameworks, board strategies, IT strategy, and existing organizational models. The Architecture Vision is an output of Phase A, not an input to the Preliminary Phase.

### Question 8
**Correct Answer: b)**
"Irrelevant" means the implementation has no relationship at all to the architecture specification—it addresses a completely different area. "Non-Conformant" means there IS a relationship but the implementation doesn't conform. This distinction is important.

### Question 9
**Correct Answer: c)**
Phase E outputs include Transition Architectures (showing intermediate states from baseline to target), an updated Architecture Roadmap, and the initial Implementation and Migration Plan. Note: the full Implementation and Migration Plan is finalized in Phase F.

### Question 10
**Correct Answer: b)**
Any artifact named "[X]/[Y] Matrix" is classified as a matrix, showing the relationship between two types of entities. The Data Entity/Business Function Matrix shows which data entities are used by which business functions.

### Question 11
**Correct Answer: b)**
Architecture Contract is a specific input to Phase G. It defines the agreed architecture that implementation projects must conform to. Phase G uses this contract as the basis for governance activities during implementation.

### Question 12
**Correct Answer: b)**
Business Skills and Methods covers understanding of business strategy, business process modeling, and financial planning. This is distinct from Enterprise Architecture Skills (which covers architecture frameworks) and Generic Skills (which covers communication, leadership).

### Question 13
**Correct Answer: b)**
The Business Footprint Diagram is a diagram that describes the links between business goals, organizational units, business functions, and services. The Organization/Actor Catalog is a catalog. The Actor/Role Matrix is a matrix.

### Question 14
**Correct Answer: b)**
The Implementation and Migration Plan is the primary output of Phase F. While the Architecture Roadmap is incrementally built through Phases B-E, the detailed Implementation and Migration Plan is specifically produced in Phase F.

### Question 15
**Correct Answer: c)**
Non-Conformant means the implementation has features that are NOT covered by the architecture specification—it has deviated from what was specified. This is different from "Irrelevant" (no relationship) and "Consistent" (moving toward conformance informally).

### Question 16
**Correct Answer: d)**
The Application/Technology Matrix belongs to the Technology Architecture domain. It maps applications to the technology platforms that support them. Despite having "Application" in the name, it is classified under Technology Architecture.

### Question 17
**Correct Answer: c)**
The Implementation and Migration Plan is an output of Phase F and is NOT an input to Phase C. Phase C inputs include the Architecture Vision, Business Architecture outputs from Phase B, and contents from the Architecture Repository.

### Question 18
**Correct Answer: c)**
Leadership and Teamwork is classified as a Generic Skill. Generic skills include communication, leadership, teamwork, and other soft skills that are broadly applicable rather than specific to architecture or business domains.

### Question 19
**Correct Answer: b)**
The Technology Portfolio Catalog is a catalog in the Technology Architecture domain. It lists the technology components used within the organization. The Environments and Locations Diagram is a diagram, and the Application/Technology Matrix is a matrix.

### Question 20
**Correct Answer: a)**
Conformant means the implementation fully conforms to all aspects of the architecture specification. "Consistent" means partial/informal conformance. There is no "Compliant" or "Aligned" level in TOGAF's defined compliance levels.

### Question 21
**Correct Answer: b)**
A key output of Phase H is a new Request for Architecture Work, which triggers a new ADM cycle when changes are needed. Phase H monitors the architecture post-implementation and identifies when significant changes warrant initiating a new cycle.

### Question 22
**Correct Answer: c)**
The Process/Application Realization Diagram is classified as a diagram. It shows how business processes are realized by applications. Artifacts ending in "Diagram" are visual representations and are classified as diagrams.

### Question 23
**Correct Answer: b)**
Application Architecture components from Phase C are a unique input to Phase D since Phase D builds upon the application architecture. Phase C doesn't have this input because it runs before Phase D (or concurrently, depending on the approach).

### Question 24
**Correct Answer: b)**
The Architecture Manager is responsible for day-to-day management of architecture development. This is distinct from the Architecture Sponsor (strategic direction), Enterprise Architect (architecture development), and Architecture Board (governance decisions).

### Question 25
**Correct Answer: c)**
The Data Entity/Business Function Matrix is a matrix artifact in the Data Architecture domain. It shows the relationship between data entities and business functions. The Data Entity/Data Component Catalog is a catalog, and the Data Dissemination Diagram is a diagram.

### Question 26
**Correct Answer: d)**
Unknown Conformance means the implementation's relationship to the architecture specification has not been assessed or is not known. This is different from "Irrelevant" (no relationship) and "Not Participating" (which is not a defined TOGAF compliance level).

### Question 27
**Correct Answer: b)**
The Architecture Definition Document is updated in each domain phase (B, C, D) to include the architecture for that domain. Each phase adds its domain-specific content to this evolving document. The Implementation and Migration Plan is specific to Phase F.

### Question 28
**Correct Answer: b)**
The Technology Standards Catalog documents agreed technology standards along with lifecycle information—whether technologies are emerging, current, contained (limited use), or retiring. This is a critical governance tool.

### Question 29
**Correct Answer: c)**
TOGAF and Architecture Framework Knowledge is classified under Enterprise Architecture Skills. This category includes architecture frameworks, architecture tools, architecture governance, and architecture modeling/patterns knowledge.

### Question 30
**Correct Answer: c)**
The Application Portfolio Catalog is a catalog that lists all applications in the organization. The Application Communication Diagram is a diagram, the Application/Organization Matrix is a matrix, and the Software Distribution Diagram is a diagram.

### Question 31
**Correct Answer: b)**
The Organizational Model for Enterprise Architecture is a key output of the Preliminary Phase. It defines the scope of the architecture function, maturity level, roles, responsibilities, constraints, and governance structures. The Architecture Vision is a Phase A output.

### Question 32
**Correct Answer: b)**
The Business Interaction Matrix shows the dependencies and communication flows between organizations, organizational units, and business functions. It identifies which parts of the organization interact and how.

### Question 33
**Correct Answer: b)**
Consistent means the implementation is moving toward conformance but on an informal basis—without a formal mandate or agreement. TOGAF's compliance levels are: Irrelevant, Consistent, Conformant, Non-Conformant, Not Participating, and Unknown Conformance.

### Question 34
**Correct Answer: b)**
The Architecture Definition Document and Architecture Requirements Specification (produced through Phases B, C, D) are key inputs to Phase E. These provide the detailed architecture work that Phase E then uses to identify implementation opportunities and solutions.

### Question 35
**Correct Answer: c)**
IT General Knowledge Skills covers broad IT knowledge including operating systems, databases, networking, and infrastructure. This is distinct from Technical IT Skills (which might be more specialist) and Enterprise Architecture Skills (which covers frameworks and methods).

### Question 36
**Correct Answer: c)**
The Networked Computing/Hardware Diagram is a diagram artifact in the Technology Architecture domain. It shows the physical network and hardware infrastructure.

### Question 37
**Correct Answer: d)**
The Implementation and Migration Plan is NOT an output of Phase A—it is produced in Phase F. Phase A outputs include the Architecture Vision, Statement of Architecture Work, draft Architecture Definition Document, and communications plan.

### Question 38
**Correct Answer: b)**
The System/Organization Matrix (also known as Application/Organization Matrix) belongs to the Application Architecture domain. It maps applications/systems to the organizational units that use them.

### Question 39
**Correct Answer: b)**
The Application/Organization Matrix is a matrix artifact in the Application Architecture domain. The Application Portfolio Catalog is a catalog, and the Application Communication Diagram and Process/Application Realization Diagram are both diagrams.

### Question 40
**Correct Answer: c)**
TOGAF defines the following compliance levels: Irrelevant, Consistent, Conformant, Non-Conformant, Not Participating, and Unknown Conformance. "Partially Compliant," "Under Review," and "Provisionally Approved" are NOT defined TOGAF compliance levels.

### Key Study Areas for Compliance Levels
Remember the six TOGAF compliance levels:
1. **Irrelevant** - No relationship to the architecture
2. **Consistent** - Moving toward conformance (informally)
3. **Conformant** - Fully conforms to the architecture
4. **Non-Conformant** - Has features NOT covered by the architecture
5. **Not Participating** - Not participating in the architecture program
6. **Unknown Conformance** - Conformance status is not known
