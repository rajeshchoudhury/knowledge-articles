## Answer Key

### Question 1
**Correct Answer:** b) Organizational Model for Enterprise Architecture
**Explanation:** The Preliminary Phase establishes the architecture capability, including the Organizational Model for Enterprise Architecture, architecture principles, tailored architecture framework, and architecture governance framework. The Architecture Vision (a) is an output of Phase A. The Architecture Definition Document (c) is built through Phases B-D. Architecture Compliance reports (d) are outputs of Phase G.

### Question 2
**Correct Answer:** c) The sponsoring organization
**Explanation:** The Statement of Architecture Work, produced in Phase A, is agreed upon by the sponsoring organization. While the Architecture Board (a) may review it, the sponsoring organization holds the approval authority as they are commissioning the work. The EA team (b) drafts it, and the CIO (d) may be a stakeholder but is not specifically identified as the approver.

### Question 3
**Correct Answer:** a) Develop Baseline Description → Develop Target Description → Perform Gap Analysis → Define Roadmap Components
**Explanation:** The standard approach within Phase B (and similarly in Phases C and D) follows a logical sequence: first understand where you are (baseline), then define where you want to be (target), identify the gaps between them (gap analysis), and finally define the roadmap components to address those gaps.

### Question 4
**Correct Answer:** b) Phase A: Architecture Vision
**Explanation:** The Request for Architecture Work is a trigger/input to Phase A. It typically comes from the sponsoring organization or is generated as a result of the Preliminary Phase. It provides the initial direction and scope for the architecture work.

### Question 5
**Correct Answer:** c) Transition Architecture
**Explanation:** Transition Architectures are outputs of Phase E (Opportunities and Solutions), not Phase C. Phase C produces the Data and Application Architecture deliverables and updates the Architecture Definition Document. Transition Architectures describe intermediate states between the Baseline and Target.

### Question 6
**Correct Answer:** b) To generate the initial complete version of the Architecture Roadmap
**Explanation:** Phase E focuses on identifying opportunities, consolidating gaps from Phases B-D, grouping them into work packages, and defining Transition Architectures. Its primary output is the initial Architecture Roadmap. Developing the Target Architecture (a) is done in Phases B-D. Governance framework (c) is established in the Preliminary Phase. Architecture Contracts (d) are finalized in Phase G.

### Question 7
**Correct Answer:** c) Phase F: Migration Planning
**Explanation:** Phase F (Migration Planning) is specifically concerned with creating the finalized Implementation and Migration Plan by prioritizing projects, performing cost-benefit analysis, and coordinating with the organization's project management frameworks.

### Question 8
**Correct Answer:** d) Phase G: Implementation Governance
**Explanation:** Architecture Contracts are outputs of Phase G. They are formal agreements between the architecture function and the implementation teams, ensuring that the implementation conforms to the architecture. While some preliminary agreements may be discussed earlier, the formal Architecture Contract is a Phase G output.

### Question 9
**Correct Answer:** c) TOGAF Library and other frameworks
**Explanation:** The Preliminary Phase takes external inputs including the TOGAF Library, other architecture frameworks (Zachman, FEAF, etc.), and organizational context. The Request for Architecture Work (b) is an input to Phase A, not the Preliminary Phase. The Architecture Vision (a) is an output of Phase A.

### Question 10
**Correct Answer:** b) Ensuring conformance of the implementation with the Target Architecture
**Explanation:** Phase G's primary purpose is to provide architectural oversight during implementation, ensuring that the solution being built conforms to the Target Architecture defined in earlier phases. Defining the Architecture Roadmap (a) is Phase E. Identifying transition architectures (c) is Phase E. Stakeholder analysis (d) is primarily Phase A.

### Question 11
**Correct Answer:** b) A change request from business or technology
**Explanation:** Phase H (Architecture Change Management) is primarily triggered by change requests — these can come from business changes, technology changes, lessons learned from implementation, or other environmental shifts. While it follows Phase G in the cycle, it is specifically triggered by the need to manage change, not merely by Phase G's completion.

### Question 12
**Correct Answer:** b) They can be done in any order or in parallel, depending on the scope
**Explanation:** TOGAF explicitly states that the ADM is flexible and that Phases B, C, and D can be performed in different orders or in parallel. The order depends on the nature and scope of the architecture engagement. Strict sequential ordering (a) is not mandated by TOGAF.

### Question 13
**Correct Answer:** c) Both Preliminary Phase and Phase A
**Explanation:** The Capability Assessment is relevant to both the Preliminary Phase (where the organization assesses its current architecture capability) and Phase A (where the capability to execute the architecture vision is assessed). It helps determine the organization's readiness for architecture work.

### Question 14
**Correct Answer:** b) Phase B: Business Architecture
**Explanation:** While all phases B through D contribute to the Architecture Requirements Specification, it is first produced as an output of Phase B. Each subsequent phase (C and D) updates and adds to this specification. Phase A produces the Architecture Vision, not the formal requirements specification.

### Question 15
**Correct Answer:** b) To prioritize projects and assign them to transition architectures
**Explanation:** In Phase F, cost-benefit analysis is performed to help prioritize migration projects and determine the optimal sequence of transition architectures. This ensures that the most valuable and feasible projects are implemented first, maximizing business value.

### Question 16
**Correct Answer:** a) A separate phase that operates at the center of the ADM cycle
**Explanation:** Requirements Management is depicted at the center of the ADM cycle diagram. It is a continuous process that interacts with all phases, ensuring that requirements are tracked, managed, and addressed throughout the entire ADM cycle. It is not limited to any single phase.

### Question 17
**Correct Answer:** b) To define the scope of the architecture effort and the principles that will guide it
**Explanation:** The Preliminary Phase is about "preparing to do architecture." Its objectives include defining the enterprise scope, establishing the architecture principles, selecting and tailoring the architecture framework, and setting up the governance structure. Creating Architecture Contracts (c) is Phase G, and validating the Implementation and Migration Plan (d) is Phase F.

### Question 18
**Correct Answer:** c) The change is managed through the Requirements Management process and may trigger Phase H
**Explanation:** TOGAF's iterative nature means that changes discovered in any phase are managed through the Requirements Management process. Significant changes may require revisiting earlier phases or triggering Phase H (Architecture Change Management). Ignoring changes (a) or immediately canceling (b) are not appropriate responses.

### Question 19
**Correct Answer:** b) Phase A: Architecture Vision
**Explanation:** The Statement of Architecture Work is a key output of Phase A. It defines the scope, approach, and deliverables for the architecture project and serves as the agreement between the architecture team and the sponsoring organization.

### Question 20
**Correct Answer:** c) Implementation and Migration Plan
**Explanation:** The Implementation and Migration Plan is an output of Phase F, produced much later in the ADM cycle. It cannot be an input to Phase B. Inputs to Phase B include the approved Architecture Vision from Phase A (a), Architecture Principles from the Preliminary Phase (b), and the Request for Architecture Work (d).

### Question 21
**Correct Answer:** b) The ADM supports revisiting prior phases when new information becomes available
**Explanation:** Iteration in the ADM context means that the process is not strictly linear. When new information is discovered in later phases, architects can revisit and update work from earlier phases. This ensures the architecture remains relevant and accurate.

### Question 22
**Correct Answer:** a) Develop Baseline Technology Architecture
**Explanation:** Phase D follows the same pattern as Phases B and C: develop the baseline, develop the target, perform gap analysis, and define roadmap components — all focused on technology. Creating the Statement of Architecture Work (b) is Phase A. Business Transformation Readiness Assessment (c) is Phase E. Finalizing the Architecture Contract (d) is Phase G.

### Question 23
**Correct Answer:** b) Preliminary Phase
**Explanation:** The Architecture Repository is established as part of the architecture capability during the Preliminary Phase. It provides the structure and storage for all architecture assets that will be produced throughout the ADM cycles.

### Question 24
**Correct Answer:** b) Architecture Vision
**Explanation:** The Architecture Vision created in Phase A serves as a guiding input to all subsequent ADM phases. It provides the high-level direction and scope that Phases B, C, D, and beyond must align with. The Implementation and Migration Plan (a) is produced in Phase F and is not available during B-D.

### Question 25
**Correct Answer:** b) Phase E identifies work packages and transition architectures; Phase F prioritizes and creates the detailed Implementation and Migration Plan
**Explanation:** Phase E focuses on identifying what needs to be done (work packages, transition architectures, solution options), while Phase F focuses on how and when to do it (prioritization, sequencing, detailed migration planning). They are complementary but distinct planning phases.

### Question 26
**Correct Answer:** b) Phase A: Architecture Vision
**Explanation:** Stakeholder mapping and management is a critical activity in Phase A. Identifying stakeholders, understanding their concerns, and managing their expectations is essential for developing an Architecture Vision that has broad organizational support.

### Question 27
**Correct Answer:** b) The ADM is a generic method that should be adapted to the specific needs of the organization
**Explanation:** TOGAF explicitly states that the ADM is a generic, flexible method. Organizations should adapt it to their specific context, including scaling phases, modifying deliverables, and adjusting the level of formality. Not every deliverable (a) is required, the Preliminary Phase (c) need not be repeated for every project, and phases can run in parallel (d).

### Question 28
**Correct Answer:** d) Phase E: Opportunities and Solutions
**Explanation:** The Consolidated Gaps, Solutions, and Dependencies matrix is a key output of Phase E. It brings together all the gaps identified in Phases B, C, and D, maps them to potential solutions, and identifies dependencies between them. This consolidation is essential for creating work packages and transition architectures.

### Question 29
**Correct Answer:** b) To evaluate the organization's readiness to undergo the change associated with the Target Architecture
**Explanation:** The Business Transformation Readiness Assessment evaluates whether the organization is ready to accept and adopt the changes required by the Target Architecture. It considers factors like organizational culture, skills, training needs, and change management capabilities.

### Question 30
**Correct Answer:** c) Phase G: Implementation Governance
**Explanation:** Phase G (Implementation Governance) focuses on overseeing the implementation and does not typically update the Architecture Definition Document. Phases B, C, and D all directly contribute to and update the Architecture Definition Document as they develop the Business, Information Systems, and Technology architectures respectively.

### Question 31
**Correct Answer:** b) It provides a high-level aspirational view of the end architecture that is used to gain stakeholder buy-in
**Explanation:** The Architecture Vision is intentionally high-level. Its purpose is to communicate the value and direction of the architecture effort to stakeholders and secure their support and commitment. It is not a detailed technical specification (a) and is referenced throughout all subsequent phases (not just Phase A).

### Question 32
**Correct Answer:** b) Merging Phase B, C, and D into a single phase for a small engagement
**Explanation:** TOGAF explicitly supports adapting the ADM, including merging phases for smaller engagements. However, removing Requirements Management (a), skipping Phase A (c), or eliminating governance (d) would undermine the fundamental integrity of the ADM process.

### Question 33
**Correct Answer:** b) Developed in the Preliminary Phase; input to all subsequent phases
**Explanation:** Architecture Principles are developed during the Preliminary Phase as part of establishing the architecture capability. They serve as guiding principles throughout the entire ADM cycle, providing input to every subsequent phase from Phase A through Phase H.

### Question 34
**Correct Answer:** b) To provide architectural oversight of the implementation
**Explanation:** Phase G ensures that the implementation projects conform to the defined architecture. It involves reviewing implementation work for compliance, handling change requests, and ensuring that Architecture Contracts are honored during the build process.

### Question 35
**Correct Answer:** b) Phase B: Business Architecture
**Explanation:** Gap analysis between the Baseline and Target Business Architecture is performed within Phase B. Each architecture domain phase (B, C, D) performs its own gap analysis specific to that domain.

### Question 36
**Correct Answer:** b) Requests for change that cannot be handled through current governance
**Explanation:** Phase H manages architecture changes. When change requests are significant enough that they cannot be handled through existing governance mechanisms, they trigger a new ADM iteration. This ensures the architecture evolves in a controlled and deliberate manner.

### Question 37
**Correct Answer:** b) Requirements Management is a continuous process that interacts with every phase of the ADM
**Explanation:** Requirements Management sits at the center of the ADM cycle and operates continuously. Requirements can be identified, modified, or retired at any point during the ADM process. It ensures that all phases remain aligned with stakeholder requirements.

### Question 38
**Correct Answer:** b) Finalized Architecture Roadmap
**Explanation:** Phase F produces the finalized Architecture Roadmap and the Implementation and Migration Plan. The Architecture Vision (a) is from Phase A. The first draft of the Architecture Definition Document (c) begins in Phase B. The Stakeholder Map (d) is from Phase A.

### Question 39
**Correct Answer:** c) Performing gap analysis between Baseline and Target Architectures
**Explanation:** Gap analysis between Baseline and Target Architectures is performed in Phases B, C, and D — not in the Preliminary Phase. The Preliminary Phase focuses on setting up the architecture capability: defining principles (a), establishing governance (b), and tailoring the framework (d).

### Question 40
**Correct Answer:** c) Return to Phase A to revise the Architecture Vision via the ADM's iterative nature
**Explanation:** The ADM's iterative nature allows architects to return to earlier phases when significant issues are discovered. If the Architecture Vision is found to be invalid during Phase C, the appropriate action is to revisit Phase A and revise it. Continuing with an invalid vision (a) would produce incorrect results. Canceling (b) is too drastic. Skipping to Phase G (d) without a valid architecture is inappropriate.
